# 输出风格

## 目标

`PM Method Agent` 的默认输出形态应当是“轻量结构化审查卡”，而不是长报告，也不是松散对话。

它需要同时满足两件事：

- 能在 IDE、命令行工具、AI 对话窗口中被快速阅读
- 保留足够稳定的方法结构，避免退化成普通聊天

## 风格原则

### 1. 结论先行

先给判断，再给依据，不要先铺垫长段背景。

### 2. 一卡一事

每条判断尽量只保留用户真正需要扫读的几个槽位：

- 判断
- 信号
- 要补

动作不需要在每条判断后重复一遍，建议收敛到卡片尾部的“先做这几步”里。

### 3. 专业但不过度正式

推荐：

- 句子短
- 结论直接
- 语气克制

避免：

- 咨询报告腔
- 过长的复合句
- 过于口语化的即时聊天表达

### 4. 决策关口可执行

决策关口要像一个可以直接作答的问题，而不是一段抽象论证。

例如：

- “问题是否已经定义清楚，可以进入方案讨论？”
- “基于现有信息，当前值得投入产品能力吗？”

## 推荐结构

一份标准输出建议包含：

1. 基础信息
2. 初步判断
3. 关键判断
4. 决策关口
5. 先做这几步
6. 待补信息

待补信息建议按类别分组，至少区分：

- 场景信息
- 现状与证据
- 决策与验证

## 不追求的风格

- 不追求完整报告
- 不追求非常强的正式公文感
- 不追求每个结论都写成长段落

这个产品更像一个“方法审查协作工具”，不是一个出咨询报告的系统。
