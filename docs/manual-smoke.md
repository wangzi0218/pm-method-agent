# 手动冒烟

## 目的

这份脚本不是替代自动化测试，而是把最近这阶段最关键的真实使用路径串起来，方便手工验证：

- 本地工具发现
- 目录枚举
- 文本搜索
- 文件读取
- 文件写入
- agent 多轮承接
- 平台写操作的审批与恢复
- 更像真人表达的交互流

## 使用方式

直接运行：

```bash
bash scripts/manual_smoke.sh
```

脚本会自动：

- 创建临时目录
- 写入最小策略文件
- 构造一组本地测试文件
- 强制关闭环境自动加载和远端 LLM，保证结果可在本地复现
- 跑一遍本地工具链
- 跑一遍多轮 agent 流程
- 跑一遍平台工具概览读取
- 跑 `tests.test_human_like_flows`
- 跑全量测试

## 适合什么时候跑

- 最近补了本地工具
- 最近改了 runtime policy / hook / tool runtime
- 最近改了 agent 多轮承接
- 准备提交一组较大改动之前
- 你想自己快速确认“现在到底能不能用”

## 当前边界

这份脚本当前偏向本地开发验证。

它还不是：

- 发布用验收脚本
- CI 工作流定义
- 网页 / 云端入口的集成测试

如果后面继续扩平台工具或混合部署，再补对应的冒烟脚本会更合适。
