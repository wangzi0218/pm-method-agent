# 方法不确定性框架

## 这份文档回答什么

`PM Method Agent` 不只是判断“信息够不够”，还需要先判断：

`当前这条需求草稿，最主要卡在哪一类方法不确定性上。`

如果这一步不先收住，后面再往问题定义、方案讨论或验证设计走，往往都会越走越散。

这份文档整理当前仓库已经接入的五类方法不确定性框架，以及它们在 agent 里的最小落地方式。

## 五类不确定性

### 1. 核心问题还没收住

典型表现：

- “真正要解决的是 A、B，还是 C？”
- “问题根源还不明确。”
- “切入点还没定。”

这类问题的关键，不是先做方案，而是先回答：

- 到底在解决哪一个问题
- 哪个问题才是当前更值得优先收敛的那个

如果这一步没做，后面很容易出现：

- 把几个相近但不同的问题揉在一起讨论
- 方案看起来都合理，但实际在打不同靶子

### 2. 用户与场景还没钉住

典型表现：

- “到底面向职场用户、学生用户，还是中老年用户？”
- “最常发生在什么高频场景？”
- “工作群、家庭群，还是一对一私聊？”

这类问题的关键，不是继续扩需求，而是先回答：

- 这轮最核心的人群是谁
- 最值得先守住的高频场景是什么

如果这一步没做，后面很容易出现：

- 一次想覆盖所有人
- 功能定位看起来很大，但没有稳定主场景

### 3. 目标与价值还没钉住

典型表现：

- “核心目标是提升时长、降低卸载率，还是提升效率？”
- “成功指标不明确。”
- “价值到底体现在哪一侧？”

这类问题的关键，是先把价值锚点定下来：

- 到底优先拉动哪个目标
- 不做会先掉哪一个结果
- 后面准备拿什么指标判断值不值得继续做

如果这一步没做，后面很容易出现：

- 优先级讨论失焦
- 方案看起来都能讲通，但没有统一判断标准

### 4. 范围与边界还没锁住

典型表现：

- “这次到底包含哪些对象？”
- “是否覆盖所有消息类型？”
- “能不能动主链路，还是只能增量优化？”

这类问题的关键，是先锁边界：

- 第一版准备先做到哪里
- 明确哪些先不做
- 明确是否允许动现有核心流程

如果这一步没做，后面很容易出现：

- 需求范围不断膨胀
- 方案从“一个优化”长成“半次重构”

### 5. 条件与约束还没摸清

典型表现：

- “底层能力是否支持？”
- “隐私合规、权限或监管有没有硬约束？”
- “和其他模块之间有没有依赖或冲突？”

这类问题的关键，是先摸清成立前提：

- 技术底座是否支持
- 合规与权限是否允许
- 系统依赖是否会改掉方案形状

如果这一步没做，后面很容易出现：

- 方案前期讨论顺畅，后期集中返工
- 需求逻辑没错，但实现条件根本不成立

## 在 agent 里的用法

当前实现里，这五类不确定性已经接入前置收敛层。

当输入明显落在其中一类时，agent 不再只泛泛地说“信息还不够”，而会先明确指出：

- 现在最大的卡点是哪一类不确定性
- 为什么这类不确定性会直接影响后面的判断
- 这一步更值得先补哪 2 到 3 个问题

也就是说，当前主线会先从：

- “缺信息”

提升为：

- “缺的是哪一类关键信息”

## 当前输出形态

当前前置收敛卡会明确给出：

- 当前判断
- 不确定性类别
- 该类别对应的 2 到 3 个聚焦追问
- 如果先按这个方向继续，下一步会怎么走

例如：

- 核心问题还没收住
- 用户与场景还没钉住
- 目标与价值还没钉住
- 范围与边界还没锁住
- 条件与约束还没摸清

## 当前边界

这套框架目前已经能覆盖一批典型产品草稿，但还不是完整的方法论引擎。

当前仍然有几个边界：

- 一条输入可能同时命中多类不确定性，但当前仍以主导类别优先
- 复杂长文本里，次要不确定性还不一定都会被完整展开
- 当前更多用于“前置收敛”，还没有把这五类直接贯穿到后续所有分析卡

## 用户怎么用

如果你是直接在 IDE 或 agent 里输入草稿，最适合的用法不是一次把所有答案都补齐，而是：

1. 先把当前模糊点直接说出来。
2. 让 agent 先判断你卡在哪一类不确定性。
3. 按那一类只补最关键的 2 到 3 个问题。
4. 再进入问题定义、决策挑战和验证设计。

更适合的输入方式例如：

- “现在我没想清楚到底在解决哪个问题。”
- “这轮核心面向的人群和场景还没定。”
- “目标到底看时长还是效率，我现在拿不准。”
- “第一版要不要覆盖全部消息类型，我还没锁边界。”
- “我不确定现有底层能力能不能支撑这件事。”

## 和主线分析的关系

这五类不确定性不是新的旁路功能，而是主线前面的一个方法层入口。

它的作用不是替代：

- 问题定义
- 决策挑战
- 验证设计

而是尽量避免在这些阶段里，用错误的问题定义继续往下走。

可以把它理解成：

`先判断现在卡在哪一类方法不确定性，再决定后面该怎么分析。`
